Intro

El trabajo práctico se encuentra alojado el la plataforma Replit, bajo el nombre gestion_de_empleados consta de 5 archivos:

  • database.py: un BackUp de la base de datos, que consiste en un script capaz de re-generar la base de datos original utilizando SQLite3.
  • gestion_empleados.db: la base de datos sobre la cual se opera con el programa.
  • main.py: el programa donde se crea la interfaz gráfica en la cual el usuario trabaja y posee dos claves: {"admin": "123456"} y {"": "") que habilita a ingresar automáticamente. Las contraseñas se encuentran con un HASH, pues el archivo se compartió a través de Replit y dicha aplicación lo hace público para todo el mundo. Esto puede saltarse si se modifica la línea 703 del código y se cambia _Inicial() por _Seleccion()
  • funcionesBdD.py: el conjunto de funciones que operan sobre la base de datos. Se hizo así porque las mismas se fueron creando y probando individualmente y en paralelo a la construcción de las pantallas en Tkinter
  • Resumen.csv: una muestra del archivo exportable que genera el programa, donde se puede observar en detalle cómo funciona la capacidad de extraer datos del programa a un archivo externo.
Consideraciones especiales

Si bien está habilitada la carga de horas extras, el sistema no carga las mismas en la pantalla Detalle del mes, esto se debe principalmente a cuestiones estéticas y de tiempo.

Como transformar todo en un único código

Si deseas combinar todo el contenido de los archivos mencionados en un solo programa, puedes seguir estos pasos:

  1. Abre cada archivo en un editor de texto o IDE, incluyendo PantallasTkinter.py, funcionesBdD.py y cualquier otro archivo relevante para tu programa.
  2. Copia y pega el contenido de cada archivo en el archivo principal de tu programa, PantallasTkinter.py. Asegúrate de pegar el contenido en los lugares adecuados, respetando la estructura y la lógica del programa.
  3. Asegúrate de resolver cualquier conflicto de nombres o duplicación de código que pueda surgir al combinar los archivos. Asegúrate de que no haya funciones, variables o importaciones duplicadas y que todo esté correctamente organizado.
  4. Una vez que hayas combinado todo el código en un solo archivo, puedes seguir los pasos mencionados anteriormente para convertirlo en un archivo ejecutable, utilizando herramientas como PyInstaller o cx_Freeze.

Recuerda revisar y probar tu programa después de combinar los archivos para asegurarte de que todo funcione correctamente y que no haya errores o incompatibilidades entre las diferentes partes del código.
